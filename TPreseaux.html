BTS SIO – 2023/2024

TP Réseaux – Configuration IP

Interconnexion de réseaux LAN via un routeur Cisco – Adressage IPv4 et passerelles

Retour aux projets

Contexte

Ce TP de réseaux a été réalisé en 1ère année de BTS SIO dans le cadre du module Réseaux. L'objectif était de configurer l'interconnexion de deux réseaux LAN distincts via un routeur Cisco simulé, en attribuant des adresses IP cohérentes à chaque machine et interface.

Les concepts clés abordés : adressage IPv4, masques de sous-réseau, passerelle par défaut et tests de connectivité par ping.

Questions / Réponses du TP

1. Quelles adresses IP avez-vous attribuées à chaque PC ?

PC2 : 172.16.200.1 | PC3 : 172.16.205.2 | PC0 : 172.17.203.3 | PC1 : 172.17.206.5

Correction : Réseau nord : 172.16.0.1 / 172.16.0.2 – Réseau sud : 172.17.0.1 / 172.17.0.2

2. Quelles adresses IP avez-vous attribuées aux interfaces du routeur ?

Interface nord : 172.16.200.254 | Interface sud : 172.17.200.254

Correction : 172.16.0.254 et 172.17.0.254

3. Avez-vous réalisé des tests de ping ?

Oui, tests réalisés via l'invite de commande et le mode simulation Cisco Packet Tracer pour vérifier la connectivité inter-réseaux.

4. Avez-vous eu besoin de configurer une passerelle par défaut ?

Oui, la passerelle est indispensable pour permettre aux machines de deux réseaux différents de communiquer. Le routeur joue ce rôle.

Correction : Oui, car les machines ne sont pas sur le même réseau.

5. Dans quels cas trouve-t-on ce type de réseau ?

Lorsque deux réseaux locaux (LAN) doivent communiquer entre eux. Le routeur, configuré avec des adresses IP spécifiques sur chaque interface, sert de passerelle.

Correction : Deux réseaux LAN interconnectés.

Technologies utilisées

Cisco Packet Tracer Adressage IPv4 Routage statique Passerelle par défaut Tests ping
